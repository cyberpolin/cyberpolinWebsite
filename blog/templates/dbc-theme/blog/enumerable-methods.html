Enumerable methods, looping on steroids

If you are researching about enumerable in Ruby you know about loops already, if you don’t have a very good grasp of loops, take a look at loops in ruby.

So you already mastered loops in Ruby, or at least you think that, i bet you when you finish this post you are going to see loops and Ruby in a different way. Enumerable module is a collection of classes, but you can think of them as a collection of loop methods.

i’m sure you remember well enough how to iter an array.

array = [“item1”, “item2”, “item3”]
array.each do |x|
  puts x
end
#output
item1
item2
item3

So that is a way of iter an array (you can do it on a hash too), and an enumerable method works the same way, in fact every object that can be iter by each can also call a enumerable method on it.

So enumerables are a way to iter arrays and hashes, but you can do a little twist to them. Why? because they help us saving a lot of keystrokes and eventually making us as developers more efficient. So now you know what they are, let's try the most used ones.


array = [“item1”, “item2”, “item3”]

array.each {|x| puts x}
#output
item1
item2
item3

Did you note that is the same example i did above? easier right?

So what if i want to know if that array have any given item?


array.each {|x| puts x if x == “item3”}

#output
item3

Or we can always do this :)


array.member(“item3”)

Great right?

Ok lets imagine we want a counter…


counter = 0
array.each { |x| counter +=1
puts counter
}

#output
1
2
3

Or do this…


array.inject{|counter, x| counter += 1}

As you can see enumerable module will enhance us as better or quicker developers, they are a lot of methods in this collection, so , my advice to you is to read the docs, and try at least everyone at least once!
