Variable from singular to general scope

Do you know what a variable is?

I guess so, but if you don’t let me explain you really fast, a name is variable, and that’s that!

Ok is not that simple… but it is, name is a variable, for example my 

 name = “Carlos”
and i’m 
 age= 35 years old. You see a name is a variable, an age is a variable so, what’s your name?, what's your age?



You see! name is a variable, age is a variable, at the end a variable is a placeholder for any value, the last two we use are different types of variable, name is a String (string of characters), the age variable is a Int (integer number) pretty simple right?



So those are the most basic types of variables, and every kind of variables goes from this two. A Int could also be a Float (floating number) for example 
age = 35.3. You see how i add a floating point in it? That is the different, but they are in the most basically sort of way the same. (Please bare with me for explaining purposes).



Any way, if i want to now my age in a ruby program i would write something like 
puts age, that would print my age, or at least the value of it so…


35.3


And if i want to know my name?
Your right!



puts name

Carlos


Now my name is not just “Carlos” so i need to update it.


name = “Carlos Gonzalez”


Well know i want to print my last name…
mmm…

i can’t do that in a simple way, but just imagine that every single character has a number which i can access it…





now if i want to print my last name i would write… ok lets make it simplier, i just want to print “G” so…



puts name[7]
Got it?

 puts name[7]+name[8]+name[9]+name[10]+name[11]+name[12]+name[13]+name[14]


That would write “Gonzalez” yeahhhhhh! So imagine we want that info in a variable.



last_name = [G,o,n,z,a,l,e,z]

The last one is an array, just like a String but with indexes. Yep those numbers are indexes so if i want to get only a G i need to access last_name[0], and so on, nice right.



But there is more!



The right way to write an array with string in it is 
last_name = [“G”,”o”,”n”,”z”,”a”,”l”,”e”,”z”] you see the cuotes is a way to tell ruby that i’m talking about strings or character, so i want to write my name and last name, but the 
last_name[0]+last_name[1]... code is not pretty at all.



But we can do this…




personal_info = [“Carlos”, “Gonzalez”]



You see what i did? Yes i have both my name and last name in a single variable! Ruby is cooooooool!!



So if i want to print my personal info i can do




puts personal_info[0]
puts personal_info[1]
puts age



That way i print all my info, but if i want to save my age to in the same variable?




personal_info = [“Carlos”, “Gonzalez”, 35]

or


personal_info = [“Carlos”, “Gonzalez”, 35.3]



So the index 0 and 1 are String, the 2 is a Int or a Float.


You see an array can have different kinds of variable even an array itself, but we see that later.


So in this way we car do something like this…



puts “Hi i’m a #{personal_info[2]} years old programer my name is #{personal_info[0]} #{personal_info[1]}


That would write “Hi i’m a 35 years old programer my name is Carlos Gonzalez”



Nice right but i want to save not just only my name, but my  address, height, weight, skills in would be difficult to access them by indexes right?



how about this…



puts “Hi i’m a #{personal_info[‘age’]} years old programer my name is #{personal_info[‘name’]} #{personal_info[‘last_name’]} i live in #{personal_info[‘country’]}, i am #{personal_info[‘height’]} height etc…”


You get the idea! cool right, that kind of variable is called a hash! and like array it can contain String, Int, Float, Array, even Hashes itself.



So how you declare a hash?




  personal_info = { “name” => “Carlos”, “last_name” => “Gonzalez”, “age” => 35, “country” => “MX”, “an_array” => [1,2,3,”Yeap an array”, “inside a hash”]}
